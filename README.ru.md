# ru-it-audit-notebook (русская версия)

Интерактивный шаблон ИТ-аудита для российских реалий: Jupyter-ноутбуки, YAML-чек-листы и Jinja2-шаблоны, которые автоматически собирают HTML/PDF отчёт для директора и технического подразделения. Проект сфокусирован на практиках ISO 27001 и требованиях ПДн/152-ФЗ, базовых контролях КИИ/187-ФЗ, резервировании, 1С/бухучётных системах и онлайн-кассах (54-ФЗ).

## Что внутри
- Чек-листы по инвентаризации, ПДн, резервированию, базовой проверке КИИ и приложениям 1С.
- Слои для расчёта баллов и приоритизации рисков по профилям (smb, fintech, outsourcer, cii-like).
- Ноутбуки, которые строят таблицы, диаграммы и собирают итоговый отчёт.
- Безопасный по умолчанию AI-слой с заглушкой `NoopAIProvider` и возможностью подключения внешнего LLM вручную.

## Как это работает
1. Заполняете YAML с ответами/инвентаризацией или используете примеры из `examples/`.
2. Запускаете CLI `ru-it-audit run-notebooks` и `ru-it-audit build-report` для сборки артефактов.
3. Получаете два вида отчёта: executive summary для руководства и техническое приложение с чек-листами и рейтингами рисков.

## Для кого
- Аудиторы и интеграторы, работающие с ПДн/152-ФЗ и резервированием.
- Внутренние DevSecOps и ИБ-команды, которым нужен быстрый формат отчётности.
- Команды, у которых в ландшафте присутствуют 1С/CRM и потенциально значимые для КИИ сервисы.

## Быстрый старт
```bash
pip install -e .[dev]
ru-it-audit run-notebooks \
  --inventory examples/inventory/sample_inventory.csv \
  --answers-dir examples/inputs \
  --out-dir .artifacts/notebooks
ru-it-audit build-report \
  --artifacts-dir .artifacts/notebooks \
  --out-html report.html \
  --out-pdf report.pdf
```

## Профессиональные услуги – run-as-daemon.ru
Проект развивается инженером DevOps/DevSecOps с сайта [run-as-daemon.ru](https://run-as-daemon.ru). Если нужен аудит инфраструктуры (1С, сервера, облака, резервирование), оценка рисков ПДн/152-ФЗ и базовых требований КИИ или регулярные отчёты для руководства, можно заказать консалтинг и сопровождение.

## Дисклеймер
Инструмент не является юридическим заключением и не гарантирует соблюдение 152-ФЗ, 187-ФЗ, 54-ФЗ или ГОСТов. Не вставляйте в отчёты реальные ПДн без законных оснований, согласовывайте выводы с юристами и ответственными за ИБ.
